Stack - Struktury danych

Nieprzerwana, dynamicznej długości struktura pobierająca najnowsze dane jako pierwsze.

Sposób działania

Stack jest jedną z najprostszych struktur danych. Jej wielkość jest zmienna, co pozwala na dodawanie i usuwanie elementów. Pamięć jest nieprzerwana, czyli wszystko pomiędzy początkiem a końcem pamięci stack jest jego elementem. Mówiąc inaczej - nie istnieje sytuacja nieistniejących lub cudzych danych w środku. Możliwe jest natomiast rozszerzenie struktury na cudzą pamięć, powodując konflikt. Korzystając z tej struktury pośród innych struktur w tym samym wycinku pamięci, należy podjąć dodatkowe działania, żeby uniknąć tych problemów.

Podstawowa wersja zawiera jedynie operacje odczytu i zapisu oraz pole z aktualną długością. Dodanie wartości powoduje wydłużenie struktury i zapis wartości na samym końcu, w utworzonym przed chwilą miejscu. Pobranie wartości powoduje skrócenie struktury i usunięcie wartości z samego końca, zwracając usunięty element.

Implementacja

Zaczynam od modułu posiadającego pamięć tworzoną w javascript oraz zmienną zawierającą długość. Limit długości stack w tym przykładzie wynosi jedną stronę pamięci WebAssembly, czyli 16384 liczb i32. Przykład można rozwinąć o konfigurację limitu i test sprawdzający, czy operacja dodania/pobrania jest możliwa. 

(module
    (memory (import "js" "memory") 1)
    (global $length (export "length") (mut i32) (i32.const 0))
)

Stack posiada tylko metodę odczytu (get) i zapisu (set) danych. Metoda get nie ma argumentów i zwraca wartość z samego końca stack, natomiast set przyjmuje argument z wartością, którą odkłada na sam koniec. 

(module
    (memory (import "js" "memory") 1)
    (global $length (export "length") (mut i32) (i32.const 0))
    (func (export "get") (result i32))
    (func (export "set") (param $v i32))
)

Do obliczenia adresu należy użyć aktualnej długości, zwracając uwagę na kolejność operacji: Aby pobrać wartość ze stack, należy zmniejszyć długość, obliczyć adres pamięci, po czym wczytać wartośc i usunąć ją z pamięci. Aby zapisać wartość do stack, należy obliczyć adres pamięci i zapisać wartość, po czym zwiększyć długość. 

(module
    (memory (import "js" "memory") 1)
    (global $length (export "length") (mut i32) (i32.const 0))
    (func (export "get") (result i32)
        global.get $length i32.const 1 i32.sub
        global.set $length

        global.get $length i32.const 4 i32.mul
        i32.load

        global.get $length i32.const 4 i32.mul i32.const 0
        i32.store
    )
    (func (export "set") (param $v i32)
        global.get $length i32.const 4 i32.mul local.get $v
        i32.store

        global.get $length i32.const 1 i32.add
        global.set $length
    )
)

Połączenie z javascript i użycie.

Klasa pośrednicząca jest prawie taka sama jak w array. 

const wasm = await fetch("./stack.wasm")
    .then(req => req.arrayBuffer())
    .then(bin => WebAssembly.compile(bin));

class WasmStack {
    #wasm;
    constructor() {
        let ready;
        this.ready = new Promise(res => ready = res);

        const wasmImports = {
            js: { 
                memory: new WebAssembly.Memory({ initial: 1 })
            }
        };
        WebAssembly.instantiate(wasm, wasmImports).then(instance => {
            this.#wasm = instance.exports;
            this.length = this.#wasm.length.value;
            ready(this);
        });
    }

    get() {
        let ret = this.#wasm.get();
        this.length = this.#wasm.length.value;
        return ret;
    }
    set(v) {
        this.#wasm.set(v);
        this.length = this.#wasm.length.value;
    }
}

Struktura jest gotowa do użycia. 

const stack = await (new WasmStack()).ready;

console.log(stack.length);
// 0

stack.set(123);
stack.set(234);
stack.set(345);

console.log(stack.length);
// 3

console.log(stack.get());
console.log(stack.get());
console.log(stack.get());
// 345
// 234
// 123

console.log(stack.get());
// memory access out of bounds

Implementacja w javascript

Ta wersja nie używa WebAssembly do działania. Tym razem obsługa jest bardziej wysokopoziomowa i część rzeczy jest wykonana automatycznie. Różnicę widać na przykład podczas prób odczytu indeksów spoza zakresu, gdzie zwracany jest undefined zamiast wywołania błędu. 

class JsStack {
    #memory;
    constructor() {
        this.#memory = new Int32Array(16384);
        this.length = 0;
    }

    get() {
        this.length--;
        const v = this.#memory[this.length];
        this.#memory[this.length] = 0;
        return v;
    }
    set(v) {
        this.#memory[this.length] = v;
        this.length++;
    }
}

const stack = new JsStack();

console.log(stack.length);
// 0

stack.set(123);
stack.set(234);
stack.set(345);

console.log(stack.length);
// 3

console.log(stack.get());
console.log(stack.get());
console.log(stack.get());
// 345
// 234
// 123

console.log(stack.get());
// undefined

Inny sposób to użycie dynamicznego odpowiednika array, dostępnego w javascript. 

class JsStack {
    #memory;
    constructor() {
        this.#memory = [];
        this.length = 0;
    }

    get() {
        this.length = this.#memory.length - 1;
        return this.#memory.pop();
    }
    set(v) {
        this.length = this.#memory.push(v);
    }
}

Zastosowanie

Stack służy do odczytywania elementów według określonej kolejności - od najnowszych do najstarszych. Struktura sprzyja szybkiemu odczytowi i zapisowi danych, ponieważ instrukcje operacji ograniczają się jedynie do obliczenia adresu pamięci i zmiany kilku wartości. Nie wprowadza też ona dodatkowych danych do swojej pamięci, przez co jej pojemność jest taka sama jak wyznaczonego wcześniej wycinka. Stack jest więc idealny do trzymania obszernych lub często pobieranych czy dodawanych danych, gdzie kolejność (od najnowszych) jest istotna. Wiele struktur danych, algorytmów a nawet całych języków programowania opiera się o stack - na przykład algorytm depth-first search albo języki WebAssembly i javascript.
