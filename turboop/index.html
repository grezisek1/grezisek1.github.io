TurboOP

Wszystkie rdzenie w garści,
bez zbędnych komplikacji.

Nie trać czasu na ręczne zarządzanie rdzeniami. To narzędzie jest stworzone dla każdego, aby w pełni wykorzystać potencjał procesora.

Czym jest TurboOP?

To biblioteka potrafiąca podzielić pracę na osobne rdzenie procesora. TurboOP kładzie nacisk na prostotę działania i łatwość użycia.

  1. Dodaj kolekcję swoich funkcji
  2. Wykonaj wybraną funkcję
  3. Odbierz rezultat
Proste, prawda?

Do czego użyć TurboOP?

Do rzeczy dozwolonych w WebWorkers

WebWorkers API, z którego korzysta TurboOP, posiada swoje ograniczenia. Nie możesz na przykład modyfikować strony, używać window ani przesłać obiektów przez referencje. Całą resztę możesz wrzucić w TurboOP.

Możliwe zastosowania

Krótka lista pomysłów kiedy użyć TurboOP.

  • Masowe zapytania do API i generowanie HTML z odpowiedzi
  • Rozkładanie zagnieżdżonych pętli (np. for w for) na osobne rdzenie
  • Konwertowanie jednostek i walut
  • Mockowanie baz danych i API
  • Kolejkowanie poleceń na później (queue)
  • Przyspieszanie metod .filter(), .map(), .forEach() i podobnych
  • Post-processing i generowanie miniaturek zdjęć i filmów
  • Konwertowanie blokujących funkcji na asynchroniczne

Jak używać TurboOP?

Prosty przykład i porównanie wydajności.

Przejdź do dokumentacji jeśli ciekawią Cię wszystkie opcje.

Twój stary kod

To jest Twój przykładowy kod, który robi przykładową pracę (matematykę). 

const doMath = (number) => {
    return number + 1;
};

const data = [1, 2, 3];
for (let i = 0; i < data.length; i++) {
    data[i] = doMath(data[i]);
}
console.log(data);

Twój nowy kod

To jest kod, który używa wszystkich rdzeni procesora. 

import TurboOP from "./TurboOP.js";

const doMath = (number) => {
    return number + 1;
};

const op = new TurboOP({
    doMathFaster: { toThread: doMath }
});

const data = [1, 2, 3];
await op.CommandBunchAndMutate("doMathFaster", data);
console.log(data);
